# 📋 产品需求文档 (PRD)：stock_analyzer 股票分析推送系统

> **产品经理**：John（BMad 产品经理）
> **日期**：2026-06-02
> **版本**：v1.0
> **定位**：盘前30秒决策简报

---

## 一、产品愿景

> **一句话定位**：每天早上8:00，钉钉收到一条消息——15只自选股+12个大盘指数+AI操作建议——30秒看完，知道今天该怎么操作。

**解决的问题**：
- 不用打开3个APP分别看A股、港股、美股
- 不用在海量新闻中人工筛选和个股相关的信息
- 不用自己对着K线图判断买卖时机——AI帮你做初步分析

---

## 二、用户画像与 JTBD

### 2.1 核心人物画像：「老陈」

- 35岁，科技公司中层
- 持有A股（五粮液、茅台等）+ 港股（腾讯、阿里）+ 美股（Apple、NVDA）
- 每天早上8点通勤，地铁上30分钟
- 想在地铁上花5分钟看完今天的操作要点

### 2.2 Jobs-to-Be-Done

| Job | 场景 | 期望结果 |
|-----|------|---------|
| **持仓速览** | 早8点，想知道昨天持仓涨跌情况 | 一眼看到每只股票涨跌幅、是否触发关注信号 |
| **大盘风向** | 想知道今天市场情绪是乐观还是悲观 | 看到A股/港股/美股三大市场的大盘指数方向 |
| **操作决策参考** | 某只股票跌了，不知道该不该卖 | AI给出基于技术面+新闻的判断：持有/卖出/观望，并说明理由 |
| **异动发现** | 持仓股放量大涨或大跌时第一时间知道 | 筛选条件触发时特别标注，AI重点分析 |
| **新闻去噪** | 不想读100条新闻，只想看和持仓相关的 | AI关联新闻与持仓股，只推送真正影响股价的消息 |

### 2.3 使用时段

```
08:00  钉钉消息到达 → 30秒速览要点
08:05  深入阅读AI分析详情
08:10  对关注的股票进入看盘软件进一步研究
09:30  A股开盘，已心中有数
```

## 三、核心功能需求

### P0 — MVP 必须实现

| 编号 | 功能 | 描述 | 验收标准 |
|------|------|------|---------|
| **P0-1** | 多市场数据采集 | 获取15只自选股行情 + 12个大盘指数 + 财经新闻 | 数据获取成功率 ≥ 90% |
| **P0-2** | AI 分析引擎 | Claude分析行情+新闻，输出每只股票的操作建议 | 分析覆盖所有15只自选股，每条建议附带理由 |
| **P0-3** | 日报生成 | 结构化晨报：摘要→大盘→自选股明细→新闻联动→操作建议 | 报告包含5个标准板块 |
| **P0-4** | 钉钉推送 | 通过Webhook发送Markdown格式消息到手机 | 消息送达，格式可读 |
| **P0-5** | 定时调度 | 每个交易日 8:00 自动执行全流程 | 8:00±1分钟触发 |

### P1 — 重要增强

| 编号 | 功能 | 描述 |
|------|------|------|
| P1-1 | 长消息拆分 | 超过18000字节时自动分段推送 |
| P1-2 | 非交易日跳过 | A股休市日不推送，港股/美股休市标注 |
| P1-3 | AI降级 | Claude API失败时推送纯数据版报告 |
| P1-4 | 海外股票中文名 | 港股美股显示中文名称 |

### P2 — 未来迭代

| 编号 | 功能 | 描述 |
|------|------|------|
| P2-1 | 智能筛选 | 全市场放量/涨跌幅/换手率/超卖发现新机会 |
| P2-2 | 历史回溯 | 查看过去N天的AI分析记录与建议准确率 |
| P2-3 | 个性化偏好 | 自定义分析风格（保守/激进） |
| P2-4 | 技术指标图表 | 报告中嵌入K线走势文字描述 |
| P2-5 | 多推送渠道 | 支持企业微信、飞书、Telegram |
| P2-6 | 公司公告 | 整合上交所/深交所正式公告 |

## 四、非功能需求

### 4.1 可靠性

| 指标 | 目标 |
|------|------|
| 整条链路成功率 | ≥ 95% |
| 单数据源故障容错 | 任一源失败不阻塞其他源 |
| AI分析降级 | Claude不可用时输出纯数据版（无AI建议） |

### 4.2 性能

| 指标 | 目标 |
|------|------|
| 全流程执行时间 | < 10分钟 |
| 数据采集耗时 | < 60秒 |
| AI分析耗时 | < 120秒 |

### 4.3 成本

| 指标 | 估算 |
|------|------|
| Claude API 单次调用 | ~$0.3-0.5（约2000-3000 input tokens + 1000 output tokens） |
| 月运营成本 | ~$10-15（约22个交易日） |

## 五、用户故事

### US-1: 每日自动推送
```
作为 持有A股+港股+美股的散户投资者
我希望 每个交易日早上8:00自动收到股票分析报告
以便 我不用手动打开多个APP就能在通勤路上快速了解持仓情况

Given 今天是A股交易日
When  系统在8:00自动触发
Then  我的钉钉收到一条包含15只自选股行情+大盘指数+新闻+AI建议的消息
```

### US-2: AI操作建议
```
作为 没有时间深入分析技术指标的投资者
我希望 AI给出每只股票的操作建议（买入/持有/卖出/观望）并附带理由
以便 我可以基于AI分析快速做出交易决策

Given 系统已获取到自选股行情和K线数据
When  AI分析引擎处理这些数据
Then  每条建议包含：操作方向、技术面理由、关联新闻（如有）
```

### US-3: 多市场统一视图
```
作为 跨市场投资者
我希望 在一份报告中同时看到A股、港股、美股的持仓情况
以便 我对全球资产配置有整体把握

Given 系统配置了A股、港股、美股自选股
When  报告生成
Then  报告按市场分组展示，含各市场大盘指数
```

### US-4: AI分析降级
```
作为 依赖每日推送做决策的投资者
我希望 即使AI分析失败，我仍然能收到纯数据版的报告
以便 我不会因为AI故障而完全失去信息

Given Claude API调用失败
When  系统检测到AI分析异常
Then  推送一份不含操作建议但包含完整行情+指数+新闻的数据报告
```

### US-5: 长消息可读
```
作为 持有15只自选股的投资者
我希望 当报告内容超过钉钉单条消息限制时，消息被合理分段
以便 我不会收到被截断的报告

Given 报告大小超过18000字节
When  系统检测到消息需要拆分
Then  消息按板块拆分（如：大盘指数一条、A股一条、港股美股一条、新闻一条）
```

### US-6: 非交易日静默
```
作为 不希望被无用消息打扰的投资者
我希望 非交易日（周末/节假日）不收到推送
以便 我的钉钉不会在不需要的时候弹出通知

Given 今天是周六或A股法定假日
When  调度器检查交易日历
Then  系统跳过本次执行，不发送任何推送
```

### US-7: 新闻关联
```
作为 关心持仓股相关消息的投资者
我希望 AI能告诉我哪条新闻可能影响我的持仓
以便 我不需要自己从海量新闻中筛选

Given 系统获取到8条财经新闻
When  AI分析新闻内容
Then  和持仓股相关的新闻被标注在对应股票下方
```

## 六、验收标准

### Checklist

- [ ] 每个A股交易日8:00（±1分钟）钉钉收到推送
- [ ] 报告包含全部15只自选股（A股8只+港股3只+美股4只）
- [ ] 报告包含全部12个大盘指数
- [ ] 每条AI操作建议附带不少于20字的理由
- [ ] 报告末尾包含「⚠️ 免责声明：以上分析仅供参考，不构成投资建议」
- [ ] 非交易日不推送
- [ ] Claude API超时/失败时推送纯数据版报告
- [ ] 单数据源失败不阻塞整条链路
- [ ] 消息超过18000字节时自动分段
- [ ] 运行日志完整记录每次执行的成败

### 一票否决红线

1. 🚫 **不得在无免责声明的情况下推送投资建议**
2. 🚫 **不得在非交易日推送消息打扰用户**
3. 🚫 **不得因单个数据源故障导致整条链路崩溃**
