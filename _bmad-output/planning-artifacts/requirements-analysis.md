## 一、利益相关者分析 — 谁是寻宝人？

### 1.1 直接用户画像

| 角色 | 描述 | 核心诉求 | 痛点 | 使用场景 |
|------|------|---------|------|---------|
| **零售投资者（核心）** | 持有A股+港股+美股的散户，工作日上午需快速了解持仓走势 | 每天早上5分钟内掌握持仓异动、大盘风向、关键新闻 | 信息过载——看盘软件资讯太多，来不及筛选；多个市场需要分别打开APP | 8:00-8:30 通勤或早餐时快速阅读 |
| **半职业交易者** | 有一定技术分析基础，关注量价关系、RSI等技术指标 | 获得带技术指标佐证的操作建议，而非泛泛而谈 | AI建议若缺乏本地化市场理解（A股政策市特征）则不可用 | 开盘前制定当日交易计划 |
| **跟单型投资者** | 信任AI判断但需要可追溯的依据 | AI建议需要附带推理过程和引用来源 | 黑箱推荐不可信，需要"为什么" | 参考AI判断辅助决策 |

### 1.2 二级利益相关者

| 角色 | 关注点 | 对项目的影响 |
|------|--------|-------------|
| **项目开发团队** | 代码可维护性、API稳定性、部署成本 | 架构决策需为后续迭代留余地 |
| **Claude API 服务商** | API 使用量、Token 消耗、回复质量 | 直接影响每日运营成本 |
| **数据源提供商（新浪/yfinance/akshare）** | 接口稳定性、反爬策略 | 最大单点故障风险源 |

### 1.3 用户旅程地图（关键路径）

```
06:30 ── 数据层启动
       ├─ 获取A股行情（新浪API）
       ├─ 获取港股/美股行情（yfinance）
       ├─ 获取12个大盘指数
       ├─ 获取财经新闻（akshare：全球快讯+东财）
       └─ 获取持仓相关公告

07:00 ── AI 分析层
       ├─ 构建Prompt（行情数据 + 新闻 + 自选股历史）
       ├─ 调用 Claude API → 生成分析报告
       └─ 格式化报告文本

08:00 ── 推送层
       ├─ 钉钉机器人推送 → "用户手机亮起"
       └─ 用户打开钉钉 → 阅读早报
```

**证据支撑**：日志文件 `logs/stock_analyzer.log` 证实了数据层执行路径——自选股查询成功（3/3），全市场筛选失败（3次重试全部断连），新闻聚合成功（5条后去重）。

### 层级 2：非功能性需求（必须做好）

| 编号 | 需求 | 指标 | 当前基线 |
|------|------|------|---------|
| NF1 | **数据时效性**：行情数据延迟 < 5分钟 | 来源确认 | 新浪API通常有15秒-3分钟延迟；yfinance延迟约15-30分钟 |
| NF2 | **执行可靠性**：整条链路成功率 ≥ 95% | 混沌测试 | 全市场扫描成功率目前 **0%**（日志证据：3/3失败） |
| NF3 | **容错性**：单数据源失败不影响其他数据 | 错误隔离 | 当前已有try/except隔离，但无降级输出策略 |
| NF4 | **成本控制**：Claude API单次分析成本 < $0.5/天 | Token计数 | 需实测待定（配置 max_tokens=4096, temperature=0.3） |
| NF5 | **推送可达性**：钉钉消息100%送达 | 推送日志 | 未实现，无重试机制 |
| NF6 | **启动时间**：从触发到推送 < 15分钟 | 运行时长 | 数据层（不含全市场扫描）+ 新闻约 3-5秒（日志推断） |

### 层级 3：约束条件

| 编号 | 约束 | 来源 | 影响 |
|------|------|------|------|
| C1 | **API Key 成本敏感性**：个人项目，预算有限 | `.env.example` `CLAUDE_API_KEY` | AI分析可能需控制长度，避免每日成本过高 |
| C2 | **数据源为免费公开API**：无SLA保障 | `a_share.py` 使用新浪JS API | 新浪/腾讯/akshare/yfinance均为免费源，随时可能变更接口 |
| C3 | **钉钉机器人仅支持单向推送**：无法交互 | 钉钉机器人Webhook规范 | 无法实现用户提问→AI回答的交互模式 |
| C4 | **A股交易时间约束**：9:30-15:00（中午休市） | A股市场规则 | 8:00推送的数据是前一日收盘数据，非实时 |
| C5 | **美股隔夜数据**：美股收盘在A股凌晨 | 时差 | 8:00推送时美股已收盘，可以包含完整美股日数据 |

## 三、竞品对标 — 藏宝图上别人的标记

### 3.1 直接竞品

| 产品 | 核心能力 | 适用人群 | stock_analyzer差距 | 差距等级 |
|------|---------|---------|-------------------|---------|
| **同花顺i问财** | AI选股+自然语言查询+图文简报 | A股散户 | 不支持海外市场；无定时推送；有APP但无钉钉集成 | 覆盖度差异 |
| **东方财富 APP「早报」** | 每日资讯简报+自选股异动推送 | A股/港股通用户 | 内容偏资讯，无个性化AI分析；无钉钉推送 | 功能深度差异 |
| **雪球「每日复盘」** | 社区讨论+组合监控+自定义推送 | 中高端投资者 | 中文名映射强于我们；但无程序化AI分析 | 维度不同 |
| **财联社「电报」** | 7x24实时快讯 | 机构投资者 | 纯资讯，无个股分析；收费产品 | 定位不同 |
| **TradingView 警报** | 技术指标触发+邮件/Webhook推送 | 全球交易者 | 无中文支持；无A股数据源；学习曲线陡峭 | 本地化差异 |

### 3.2 差异化定位

```
stock_analyzer 的核心差异化：
    1. A股 + 港股 + 美股 三市场统一晨报（同花顺/东财做不到跨市场）
    2. Claude AI 原生分析（非关键词匹配，是真·LLM推理）
    3. 钉钉推送（触达效率高于APP推送，更适合工作场景）
    4. 全自动化零人工干预（雪球/财联社需要编辑团队）
```

## 四、风险清单

### 4.1 技术风险

| 风险 | 概率 | 影响 | 等级 | 证据 | 缓解策略 |
|------|------|------|------|------|---------|
| **新浪 API 全市场扫描不稳定** | 高 | 高（智能筛选不可用） | 🔴 严重 | 日志确证：2026-06-02 3次重试全部 "Remote end closed connection without response" | 降级——全市场扫描失败时仅分析自选股；增加备用源（腾讯API/akshare）；缩短分批大小 |
| **新浪 API 自选股同样可能故障** | 中 | 高（A股数据全缺） | 🟡 高 | 日志显示自选股3/3成功，但同网域，有连带风险 | 增加腾讯行情API作为A股备用源 |
| **yfinance 限流/封IP** | 中 | 中（海外数据缺失） | 🟡 高 | yfinance为社区逆向工程，无官方保障 | 增加缓存TTL；考虑单次请求间隔 |
| **akshare 版本兼容** | 中 | 中（新闻缺失） | 🟡 中 | akshare迭代频繁，API参数变更常见 | 在requirements.txt中锁定版本号 |
| **Claude API 调用超时/失败** | 低 | 高（AI分析缺失） | 🟡 高 | 依赖网络+Anthropic服务可用性 | 重试机制+超时兜底（输出简版无AI报告） |
| **钉钉Webhook 限频** | 低 | 中（推送失败） | 🟢 低 | 钉钉机器人有每分钟20条限制 | 合并为单条消息推送 |

### 4.2 合规风险

| 风险 | 说明 |
|------|------|
| **投资建议合规性** | 中国证监会规定：未经许可提供证券投资建议可能违法。AI"操作建议"需以"仅供参考"免责声明为前提 |
| **数据源使用条款** | 新浪/yfinance/akshare的数据不可商用。本系统若仅限个人使用则合规，如扩大范围需注意 |
| **钉钉消息合规** | 推送内容不得包含虚假信息、内幕信息或诱导性表述 |
